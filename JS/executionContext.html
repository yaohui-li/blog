    # Javascript的执行上下文

    执行上下文是当前执行代码的执行环境, 它决定了当前执行代码所能使用的变量和函数, 每个上下文都有一个关联的变量对象(variable object 简称VO), VO中定义了所有变量和函数.

    执行上下文主要包括三种:

    1. 全局执行环境
      全局执行环境为最外层的上下文, 根据ECMAScript实现的宿主环境. 表示全局上下文的对象可能不一致. 在浏览器中是window对象, NodeJs中global object对象

    注意

    💡 var在全局中变量会成为window对象的属性 let和const则不会,但是解析作用域的效果是一样的

    1. 函数执行环境
      函数执行环境是函数执行之前产生

    2. eveal(不推荐使用, 基本可以忽略)

    从一个例子分析JavaScript的执行上下文的生命周期

      var color = blue;
  function changeColor() {
    var anotherColor = red;
    function swapColor() {
      var tempColor = anotherColor;
      anotherColor = color;
      color = tempColor;
    }
    swapColor();
  }
  changeColor();

    具体步骤:

    1. 生成存放执行上下文的堆栈, 然后生成一个全局的执行上下文, 将它放入堆栈中(也叫压栈)
    2. 当遇到changeColor调用执行时, 在此之前将创建一个changeColor函数的执行上下文, 将放入堆栈中, 此时changeColor上下文在全局上下文上面,即在堆栈的栈顶
    3. 执行changeColor代码时, 遇到了swapColor函数调用执行, 在执行之前将创建swapColor函数的执行上下文, 将它压栈, 此时swapColor上下文在堆栈的栈顶
    4. 执行完swapColor代码后 swapColor上下文被弹出栈, 然后changeColor出栈. 全局上下文只有在关闭网页或浏览器的才销毁
