    # 作用域和作用域链

    1. 什么是作用域

    作用域是当前执行代码可以访问的标识符(变量), 在执行上下文产生的过程中, 就形成了当前执行代码的作用域.

      var globalVar = 'global';
  function fn() {
    var localVar = 'local';
    say(localVar);
  }
  function say(value) {
    var name = 'My is';
    console.log(`${name} ${value}`);
  }
  fn();

    从上面的例子中分析作用的产生:
    1. 产生全局执行环境同时产生全局作用域. golbal Object对象中存放了window对象等全局能访问的变量和函数. 例如: globalVar变量 fn函数 say函数;
    2. 当遇到fn()时 会生成fn的执行上下文, 从而生成一个fn的局部作用域. Active Object中存放 localVar变量
    3. 遇到到say函数时 又生成一个say的执行上下文, 生成一个say的局部作用 Active Object中存放 arugments的参数 value 和 name

    TIP

    外层作用域不能访问内层作用域的变量和函数

    1. 什么是作用域链

    作用域链是在生产执行上下文时生成了一条当前执行环境有权访问的所有变量和函数的有序访问.

      var color = 'red';
  var name = 'xiaoming';
  var getColor() {
    var color = 'blue'
    console.log(`${name} is dress is ${color}`)
  }
  getColor(); // xiaoming is dress is blue

    分析上述代码:

    1. 生成全局作用域 global scope 其中生成了color和name全局变量以及函数getColor
    2. 遇到getColor函数调用时, 生成getColor的执行上下文 , 同时将全局作用域的引用执行getColor的active Object上. 因此 getColor中获取了到了name变量.

    注意

    函数里面变量的查找遵循就近原则, 在当前作用域下有,就用当前变量, 如果没有就查找外层, 如果外层没有,就找外层的外层,依次递进向外查找. color变量没有获取全局变量的 而是函数内部的color变量, 如果去掉 var color = 'blue' 则将获取全局变量color的值'red'

      var action = 'say';
  function fn() {
    var action = 'read'
    var name = 'xiaolong'
    function say() {
      console.log(`${name} is ${action}ing`)
    }
    say()
  }
  fn(); // xiaolong is reading
  // 如果去掉 fn()里面的 var action = 'read'
  // 结果将会是 xiaolong is saying
